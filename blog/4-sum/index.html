4 Sum

LeetCode #18


Description

Given an array S of n integers, are there elements a, b, c, and d in S such that a + b + c + d = target? Find all unique quadruplets in the array which gives the sum of target.

Note: The solution set must not contain duplicate quadruplets.

For example, given array S = [1, 0, -1, 0, -2, 2], and target = 0.

A solution set is:
[
  [-1,  0, 0, 1],
  [-2, -1, 1, 2],
  [-2,  0, 0, 2]
]


My Thoughts

It is similar to 3 Sum problem. I think I will also make a helper function twoSum and use two for loops to calculate two numbers first. Then call twoSum with rest of numbers and target = target - sum of two numbers.

However, it works but it runs too slow.

So, I think I have to take advantage on sorted array. And also change my solution into a recursive function which can handle N Sum problem, and it considers about following condition:

  • If number of nums is less than N or N < 2, it means that the remaining nums cannot come up with a result, so just return.
  • If N == 2, then do two sum as usual with two pointers.
  • Else, run for loop for all remaining nums:
    1. To speed up, I have to take advantage of sorted array. If target < nums[i] * N or target > nums[-1] * N, it is impossible to get N numbers which sum equals to target, so I can break the loop.
    2. Take advantage of sorted array again, I can just process index where nums[index-1] != nums[index], cause I don’t have to compute the same answer. And in this condition, I can call recursive function with nums = nums[i+1:], target = target - nums[i], and N = N-1


Solution

class Solution(object):
    def fourSum(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: List[List[int]]
        """
        results = []
        
        def findNSum(nums, target, N, result):
            if len(nums) < N or N < 2:
                return
            
            # two Sum
            if N == 2:
                left = 0
                right = len(nums)-1
                while left < right:
                    if nums[left] + nums[right] == target:
                        results.append(result + [nums[left], nums[right]])
                        left += 1
                        right -= 1
                        while left < right and nums[left] == nums[left-1]:
                            left += 1
                        while left < right and nums[right] == nums[right+1]:
                            right -= 1
                    elif nums[left] + nums[right] < target:
                        left += 1
                    else:
                        right -= 1
            else:
                for i in range(0, len(nums)-N+1):
                    if target < nums[i]*N or target > nums[-1]*N:
                        break
                    if i == 0 or i > 0 and nums[i-1] != nums[i]:
                        findNSum(nums[i+1:], target-nums[i], N-1, result+[nums[i]])
            return
        
        nums.sort()
        findNSum(nums, target, 4, [])
                    
        
        return results
