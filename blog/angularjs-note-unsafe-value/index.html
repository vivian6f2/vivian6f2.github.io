Notes for AngularJS

Error: unsafe value used in a resource URL context

I met this error when I tried to set iframe src attribute. And after searching this error message, I got this answer.

So let me just simplify this answer into my own word. In the component that I want to set the url for iframe src:

  1. Import SafeResourceUrl and DomSanitizer

    import { SafeResourceUrl, DomSanitizer } from '@angular/platform-browser';

  2. Add DomSanitizer into contructor

    constructor(private sanitizer: DomSanitizer) {
}

  3. Use SafeResourceUrl for the variable which if for iframe src

    this.safeUrl = sanitizer.bypassSecurityTrustResourceUrl('myIframeSrc');

After doing above steps, there may not be any other error for unsafe value.
